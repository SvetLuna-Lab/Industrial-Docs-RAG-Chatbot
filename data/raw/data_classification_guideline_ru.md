# Руководство по классификации данных

Документ-черновик.  
Цель: задать базовые уровни классификации данных и правила обращения с ними  
для дальнейшего использования в RAG-системах и внутренних сервисах.

## 1. Уровни классификации

Рекомендуется использовать не менее четырёх уровней:

1. **Публичные данные**  
   - информация, официально публикуемая наружу;
   - примеры: пресс-релизы, маркетинговые материалы.

2. **Внутренние данные**  
   - информация для использования внутри компании;
   - примеры: внутренние инструкции общего характера, методички.

3. **Конфиденциальные данные**  
   - данные, раскрытие которых может нанести ущерб компании;
   - примеры: коммерческие условия, часть технической документации, схемы сети.

4. **Строго конфиденциальные данные**  
   - критичная информация с повышенными требованиями защиты;
   - примеры: ключи шифрования, пароли, детальные схемы ИТ/ОТ-инфраструктуры.

## 2. Общие правила работы с данными

1. Уровень классификации должен назначаться **при создании документа** либо при его первичной регистрации.
2. При копировании/пересылке документа уровень классификации должен сохраняться.
3. Запрещается понижать уровень классификации без согласования с ответственным владельцем данных.

## 3. Классификация для RAG-систем

При подготовке корпуса для RAG:

1. В индекс по умолчанию разрешается включать:
   - публичные данные;
   - внутренние данные (при наличии базовых средств контроля доступа).

2. Для конфиденциальных и строго конфиденциальных данных требуется:

   - отдельный индекс и/или отдельный контур;
   - аутентификация и авторизация пользователей;
   - журналирование обращений к этим данным.

3. Категории, которые **не должны** попадать в общий RAG-индекс:
   - данные аутентификационных систем (пароли, ключи, токены);
   - персональные данные без явного юридического основания на обработку;
   - любые материалы, прямо запрещённые внутренними или внешними регуляторами.

## 4. Обозначение уровня в документах

Рекомендуется указывать уровень классификации:

- в начале документа (заголовок / шапка);
- в системах документооборота — отдельным полем (например, `classification_level`).

Пример обозначения:

- `[INTERNAL] Регламент обслуживания насосного агрегата Н-250`
- `[CONFIDENTIAL] Схема сетевой инфраструктуры ПНС-1`

## 5. Ответственность

- Владелец данных отвечает за корректное определение уровня классификации.
- Пользователи отвечают за соблюдение правил обращения с данными:
  - запрет на несанкционированное копирование;
  - запрет на пересылку во внешние системы без согласования.
